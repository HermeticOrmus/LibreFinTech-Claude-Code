# Intermediate Learning Path - Financial Technology

## Overview

This intermediate path covers Financial Technology concepts and practices, progressing from fundamentals to practical application with Claude Code plugins.

## Prerequisites

- Completed beginner path\n- Basic ledger implementation\n- Payment integration experience

## Modules

### Module 1: KYC/AML & Compliance

#### Concepts
- Identity verification workflows\n- Sanctions screening\n- Transaction monitoring\n- Regulatory reporting basics\n- PCI DSS requirements

#### Hands-On Exercise
Apply the concepts using the relevant plugins. Start simple, build complexity gradually.

#### Key Takeaways
- Understand the fundamentals of KYC/AML & Compliance
- Know when to apply these patterns
- Create basic implementations confidently

---

### Module 2: Risk Management

#### Concepts
- Credit risk scoring\n- Fraud detection rules\n- Risk model basics\n- Real-time risk assessment\n- Anomaly detection patterns

#### Hands-On Exercise
Build on Module 1 with more advanced techniques. Focus on quality and reproducibility.

#### Key Takeaways
- Integrate multiple concepts effectively
- Apply established best practices
- Evaluate and improve existing work

---

### Module 3: Financial APIs

#### Concepts
- Open Banking standards\n- PSD2 compliance\n- Bank API integration\n- Webhook handling\n- API security (OAuth, mTLS)

#### Hands-On Exercise
Combine everything into a comprehensive project demonstrating full mastery.

#### Key Takeaways
- Synthesize all module concepts
- Make informed architectural decisions
- Create production-quality implementations

## Assessment

- [ ] Complete all module exercises
- [ ] Build a project combining all concepts
- [ ] Review against skill file patterns
- [ ] Identify areas for deeper study

## Next Steps

Continue to **advanced** for trading systems, real-time settlement, and advanced risk.
